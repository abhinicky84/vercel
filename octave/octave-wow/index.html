🔄 Ways of Working (Proposed Lifecycle)

A predictable, highly automated 10-step software development lifecycle (SDLC) designed to run features in parallel while ensuring complete quality control.

1
Draft User Stories WPP & Ogilvy Team

Initial technical specifications, acceptance criteria, and operational scopes are thoroughly drafted as actionable tasks inside Wrike.

2
Octave Product Approval Octave Product Team

Octave's product stakeholders review the specifications in Wrike and formally sign off on requirements, setting clear expectations before any development begins.

3
Parallel Feature Development & Local Unit Testing WPP Dev Team

WPP developers pick up the approved user story, build the feature in local feature branches, and write rigorous unit tests to verify initial stability.

4
Commit to GitHub & Trigger Dev Environment Build Optional Tommy Review

We commit code to the designated branch, which triggers the automated CI/CD pipeline to deploy onto the Dev environment. This serves as an optional code review gate for Tommy.

5
Internal Dev Verification & QA Hand-off WPP Dev Team

The team validates the implemented feature on the Dev environment to confirm functionality meets Wrike acceptance criteria, marking the issue as ready for QA.

6
Trigger QA Environment Build WPP Dev Team

The CI/CD pipeline is automatically triggered to deploy the verified packages to the isolated QA environment.

7
QA Verification & Iterative Bug Fixing WPP QA & Dev

Our QA team thoroughly tests the environment. If issues are found, they are logged transparently in Wrike; developers immediately patch and re-deploy to the QA environment for final pass validation.

8
Trigger Staging Environment Build WPP Dev Team

Once QA signs off, code branches are compiled and automatically deployed to the Stage/Pre-production environment.

9
UAT Request & Sign-off Gate Tommy & Octave Product

We request formal User Acceptance Testing (UAT) and architectural checks from Tommy and the Octave product team on the Stage environment.

10
Production Release & Delivery Joint Release Management

With all reviews complete, the certified build is promoted to the live Production environment in coordination with Tommy's release timeline.
