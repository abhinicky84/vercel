02. Strategic Options Comparison

Evaluating three paths structured by organizational complexity and ROI. Since Octave possesses an established mainland entity, the recommended hybrid approach offers a scalable, compliant, and cost-efficient path.

Option A

Chinafy SaaS Automation Overlay

Lightest Lift - No Code Solutioning Time: Low

Chinafy works as a premium front-end automated proxy layer. When mainland traffic queries octave.com, they are seamlessly routed to a real-time optimized, cached replication of the global page. The system automatically identifies blocked components (AWS/Vidyard) and swaps them with fast, China-accessible alternative CDNs, requiring zero engineering effort.

Strategic Advantages
  • Near-zero engineering footprint and incredibly fast to deploy.
  • No manual synchronization required—auto-caches Contentful updates instantly.
  • Automates complex Vidyard conversion paths without manual video uploads.
Key Considerations
  • Premium pricing structure starting around $2K-$3K/month.
  • "Black box" mechanics limit direct engineering oversight and custom layout tweaks.
  • Adds a third-party gateway directly in front of primary hosting infrastructure.
Option B — Recommended Pathway

Cloudflare China Network + Onshore Media Mirroring

⭐ Highly Recommended Solutioning Time: Medium

This strategy maps directly to your existing tech stack and regional corporate footprints. First, activate Cloudflare Enterprise China Network (operating onshore nodes via JD Cloud). Next, mirror specific hero loop videos and audio assets onto **Azure operated by 21Vianet** (the secure, isolated onshore China Azure cloud). Use Azure Traffic Manager with geo-routing logic to serve mainland users native onshore video players while keeping Vidyard active globally.

Strategic Advantages
  • Resolves both global site latency and localized media delivery challenges.
  • Capitalizes directly on existing corporate relationships and experience with Azure.
  • Simplifies ICP filing procedures (non-commercial B2B filing only).
  • Maintains full codebase and tracking unity (GA4, Google Tag Manager).
Key Considerations
  • Requires establishing a separate, isolated Azure 21Vianet account inside China.
  • Requires minor editorial adjustments for dual asset uploads (Vidyard and Azure China).
  • Frontify media CDN directories must also be proxied or mirrored onshore.

Option C

Dedicated Subdomain Infrastructure (cn.octave.com)

Heaviest Technical Lift Solutioning Time: High

The enterprise isolation strategy. Setup cn.octave.com as an independent environment deployed entirely on Azure 21Vianet or local providers like Alibaba Cloud. While global editor structures remain synchronized via central Contentful CMS APIs, the delivery framework, CDN architecture, and video storage systems are managed as a separate, fully compliant mainland pipeline.

Strategic Advantages
  • Zero complex DNS or routing logic required.
  • Isolated regulatory footprint; ICP compliance is cleanly contained within cn.octave.com.
  • 100% native speeds inside China with no foreign dependencies.
Key Considerations
  • High upfront CAPEX and increased overhead for dev teams.
  • Risk of domain split impacting search engine optimization authority.
  • Requires custom data bridge development to unify performance dashboards.

03. Deployment Roadmap

Leveraging Octave's strong local legal presence, we have mapped out a phased execution structure designed to achieve rapid compliance while minimizing operational drag.

📋
CRITICAL FIRST STEP

Onshore ICP Filing (Bei'An 备案)

To route traffic through onshore mainland servers (including Cloudflare Enterprise China or local Azure instances), Octave must secure an **ICP Filing** registered to its Chinese business entity. This is an achievable, non-commercial registration that represents the single most critical prerequisite. Once approved, the filing number must be cleanly displayed in the footer of all /zh pages.

01 Phase 1: Foundation
ICP Filing & Sandbox Setup

Initiate the B2B ICP filing (Bei'An) process using Octave's local business credentials. Simultaneously set up the Azure China 21Vianet trial environments.

Objective: File documentation & prepare cloud environments.
02 Phase 2: Execution
Option B Deployment

Mirror critical hero videos and assets onto local Azure storage. Configure routing parameters in Traffic Manager and active Cloudflare China Network nodes.

Objective: Complete media loading speed optimizations.
03 Phase 3: Scale
Traffic Evaluation

Monitor localized load speeds and audience metrics. Only consider moving to a dedicated subdomain (Option C) if local traffic scales significantly.

Objective: Achieve optimal speed-to-investment ratio.
