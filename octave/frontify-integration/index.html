Feasibility Analysis

Real-Time Frontify Previews in Contentful Editorial Area

Executive Summary: Yes, this can be done immediately. We are replacing the highly inefficient, blind-linking copy-paste workflow with a native visual asset browser directly inside the Contentful entry editor. Authors gain instant visual confirmation of Frontify DAM assets at the exact moment of selection.

The Editor Experience Bottlenecks

  • Blind DAM Linking: Content authors currently paste raw, long Frontify CDN strings into text fields with zero visual confirmation.
  • Broken Web Layouts: Because authors cannot verify aspect ratios during selection, mismatched images slip into production.
  • Duplicate Asset Overhead: Authors download images from Frontify to their desktops and re-upload them to Contentful, breaking the single-source-of-truth brand architecture.

The Visual Integration Solution

  • Native Asset Browser Overlay: Leverages Frontify's asset finder within Contentful so authors browse, filter, and search DAM libraries directly.
  • Inline Active Previews: Shows instant visual thumbnails, size parameters, and file type badges inside Contentful prior to publishing.
  • Revision-Safe CDN Referencing: Employs Frontify's revision-stable CDN links, ensuring web pages dynamically point to updated brand assets safely.

Technical Specification

Rich Field Schema & Frontend Rendering Model

To support responsive layouts and prevent layout shifts on the live website, we bind a robust metadata payload to the custom Contentful field rather than simple URL strings.

Standard Frontify Metadata Payload

Upon asset selection, the custom Contentful integration app captures and stores a rich JSON metadata payload containing the visual asset specifics:

{ "assetId": "frontify_image_440a7", "title": "Octave Aria Platform Hero Mockup", "previewUrl": "https://octave-cdn.frontify.com/api/preview/...", "cdnUrl": "https://octave-cdn.frontify.com/cdn/...", "width": 1200, "height": 800, "focalPoint": "[0.5, 0.5]" }

Performance-Optimized Rendering Component (Vue.js)

We implement this standard component structure to automatically read the custom payload, reserving pixel-perfect layout coordinates before the image loads:

<template> <div :style="{ aspectRatio: `${payload.width} / ${payload.height}` }" class="ratio-box"> <img :src="payload.cdnUrl" loading="lazy" :alt="payload.title" /> </div> </template>
